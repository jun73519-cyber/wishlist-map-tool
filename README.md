# 行きたい場所マップ帳

行きたい場所を集めて、**気になる → 計画中 → 予約済 → 訪問済**の段階で管理し、
旅の情報・メモ・写真を 1 画面で見渡せる 4 ペインのデスクトップ Web アプリです。

採用管理サンプル（workspace-ui-kit）を土台に、旅行ドメインへ作り変えて作っています。

## 画面構成（4 ペイン）

| ペイン     | 役割                                                                              |
| ---------- | --------------------------------------------------------------------------------- |
| **Pane 1** | エリア（国内 / 海外 → 北海道・東北 / 関東 / 関西 / アジア / ヨーロッパ …）の絞り込み |
| **Pane 2** | 行きたい場所リスト。検討ステージ（気になる/計画中/予約済/訪問済）で分類。ドラッグで段階を移動、不要になった場所は「見送り」へ |
| **Pane 3** | 場所ダッシュボード（読む場所）。基本情報 / リンク・連絡先 / 旅行メモ / 検討フロー |
| **Pane 4** | ステージ別の詳細（編集の本拠地）。交通手段・同行者・★行きたい度・メモ・写真資料   |

> `docs/screenshot-workspace.png` は旧サンプル（採用管理）の画像です。最新の画面で撮り直すと実態に合います。

## 起動する

```bash
npm install
npm run dev
```

ブラウザで `http://localhost:3000`（使用中なら 3001）を開くと画面が表示されます。

## 自分の「行きたい場所」に作り変える

データとラベルはコードから分離してあるので、以下を書き換えるだけで中身を差し替えられます。

| 変えたいもの                       | 編集するファイル                                              |
| ---------------------------------- | ------------------------------------------------------------- |
| 行きたい場所（サンプルデータ）     | `data/candidates.json`                                        |
| エリア（Pane 1 の階層）            | `data/positions.json`                                         |
| アプリ名・アイコン                 | `data/workspace.json`                                         |
| ステージ名・評価軸・セクション見出し | `lib/labels.ts`                                               |
| 色・角丸・余白などのデザイントークン | `app/globals.css`（`@theme` の CSS 変数）                     |

> 内部のキーや型（`screening`/`first`/… や `Candidate` 等）は採用管理サンプル由来の名前のまま据え置き、**表示名とデータだけ**を旅行ドメインに差し替える方針です。評価は 1 軸「行きたい度」（`wishLevel`）に変更しています。

## ディレクトリ構成

```
app/                Next.js App Router の画面エントリ（page.tsx / layout.tsx / globals.css）
components/
  ui/               shadcn の UI 部品（Button, Card, Dialog 等）
  primitives/       インライン編集の独自プリミティブ（Inline*Field 等）
  workspace/        4 ペイン本体（Pane1〜4 と関連ダイアログ）
data/               サンプルの種データ（JSON）
hooks/              React のカスタムフック
lib/                型定義（zod スキーマ）・ラベル・派生計算・ユーティリティ
__tests__/          テスト（Vitest）
```

## 開発コマンド

| コマンド               | 役割                          |
| ---------------------- | ----------------------------- |
| `npm run dev`          | 開発サーバー起動              |
| `npm run build`        | 本番ビルド                    |
| `npm run lint`         | ESLint チェック               |
| `npm run test`         | スモークテスト（Vitest）      |
| `npm run format`       | Prettier で整形               |
| `npm run check:radius` | 角丸ドリフト検出（独自スクリプト） |

> 開発サーバー稼働中に `npm run build` を走らせると `.next` を奪い合って
> Turbopack のキャッシュが壊れることがあります。ビルド検証時は dev を一旦止めてください。

## 技術スタック

- **Next.js 16** / **React 19** / **TypeScript**（strict）
- **Tailwind CSS v4**（`app/globals.css` の `@theme` で CSS 変数を一元管理）
- **shadcn/ui**（`base-nova` スタイル / `@base-ui/react` ベース）
- **lucide-react**（アイコン）/ **zod**（実行時の型検証）/ **@dnd-kit**（ドラッグ＆ドロップ）

## UI を編集するときのメモ

このリポジトリには UI 作業の規律をまとめた `designing-workspace-ui` スキルが同梱されています。
ペインの責務・色や角丸の階層・インライン編集の保存規約などを守るためのものです。
AI に UI 変更を頼むときは「designing-workspace-ui スキルを使って」と添えると、規律に沿った変更が返ってきます。

- 4 ペイン構造を尊重する（Pane 3 = 読む場所 / Pane 4 = 編集の本拠地）
- 色は役割で名前付けされたトークン（`bg-primary` 等）を使う
- フィールド編集は `components/primitives/Inline*` を再利用する
